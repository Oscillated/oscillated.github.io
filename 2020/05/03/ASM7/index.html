白洪欢《汇编语言程序设计基础》笔记7

零标志 ZF

CF ZF都在FL标志寄存器中，

ZF记录刚刚的计算结果是否等于0

ZF=1 结果为0

ZF=0 结果不为0

je 与 jz等价，因为跳转的依据是ZF=1

jne 与 jnz等价，跳转的依据是ZF=0

je - 两数是否相等，实际通过一个相减操作来判断

mov指令不影响任何标志位

符号标志 SF

针对符号数运算，判断当前计算有无产生负数

运算结果的最高位为1，是负数，SF=1

js (jump if sign flag) 如果符号标志为1则跳转

jns (jump if no sign flag)

溢出标志 OF

正+正=负 或 负+负=正 的情况

正负相加永不溢出

产生溢出，OF=1

无溢出，OF=0

jo(jump if overflow)

jno(jump if no overflow)

非符号数加法也会有溢出，但是标志是CF：

cpu在设置各个标志位的时候，既把所有的数看作非符号数运算，又将其看成符号数运算，才可能设定各标志位的状态

奇偶标志符PF

parity flag

统计运算结果低8位

PF=1表示有偶数个1

PF=0表示有奇数个1

jp 当PF=1时跳转，jp也可以写成jpe(jump if parity is even)

jnp 当PF=0时跳转，jnp也可以写成jpo(jump if parity is odd)

用途

串口的串行通信时的奇校验和偶校验

串行通信：数据位的传送，按位顺序进行，最少只需一根传输线即可完成；成本低但传送速度慢。串行通讯的距离可以从几米到几千米

奇校验：数据位+校验位合起来，1的个数必须是奇数

偶校验：数据位+校验位合起来，1的个数必须是偶数

‘A’=0100 0001B 若采用奇校验，则把它变成1100 0001B再发送

对方接收到这个值保存到AL中，验证：

辅助进位标志符AF

auxiliary flag

统计低4位有无向高4位产生进位或者借位

只要第3位向第4位进位或借位，AF=1

无相关跳转指令，但有相关的BCD码计算指令，如daa(decimal adjust for add), aaa

BCD码

binary coded decimal

时刻19:11

计算机保存为19h:11h

优点

便于把十位和个位分离开

19h >> 4 -> 01h 十位

19h & 0Fh -> 09h 个位

如果在十进制下提取各位数字：

19 / 10 = 1 十位

19 % 10 = 9 个位

但是除法麻烦、速度慢，不如右移

缺点

十六进制表示下加法会有错误，需要调整

11h+09h=1Ah 需要调整，只要+6即可 = 20h (10+6=16可以构成进位);AF=0

18h+09h=21h 需要调整，只要+6即可 =27h;AF=1

8h+9h= 16h + 1h = 11h = 10h+1h 逢十六进一 -> 逢十进一，补6

daa

  1. 判断个位有无大于A
  2. 判断AF是否为1

所做的调整就是+6

出现一个是，使用daa调整al中的值

小结

CF ZF SF OF AF PF 6个状态标志 先计算，反映计算后的状态，再通过跳转影响程序

DF TF IF 3个控制标志 先改变再影响CPU

方向标志DF

Direction flag

影响CPU对字符串的操作方向

DF=0时为正方向（由低地址到高地址）

DF=1时为反方向（由高地址到低地址）

cld指令使DF=0，clear direction

std指令使DF=1，set direction

中断标志 IF

Interrupt flag

IF=1时允许硬件中断

IF=0时禁止硬件中断

cli指令使IF=0

sti指令使IF=1

中断：计算机硬件向CPU发出中断请求

硬件中断：由某一event如敲键触发，cpu自动调用

软件中断：代码内写好的int指令实现

时钟中断：每当程序运行1/18秒,则cpu会在此处插入一条int 8h指令，发出呼叫请求cpu暂停，cpu收到呼叫后对cpu内部的变量n自增

程序运行速度很快用户无法感受到延迟，对于操作系统实现多任务起着不可或缺的作用，操作系统需要在int 8h中做好时间分配。cpu同一时刻实现一个程序，分时执行，控制权切换是由于int 8存在，A在执行过程中一定有int 8，交出控制权给B，DOS同一时间只能运行一个程序

同一程序两部分运行——线程thread

不同程序运行——进程process

硬件中断有便利也有缺陷，可能会影响程序运行效率

用cli和sti把一段代码包围起来可以达到该段代码在执行过程中不会被打断的效果：

windows 中 cli 和 sti 为特权指令，用户无法使用这两条指令，除非在.sys驱动程序中使用

跟踪/陷阱标志 TF

Trace/Trap flag

当TF=1时，CPU会进入单步模式single-step mode，CPU在每执行完一条指令后,会自动在该条指令与下条指令之间插入一条int 1h指令并执行它。

硬件支持单步中断的发生，因此调试器可以调试代码 用户代码与调试器代码轮流获得控制权

可以实现反调试，int 1h后等待用户键盘输入进行下一步操作

要修改TF=1必须用pushf和popf指令（TF太冷门，没有专门设计）

clt stt指令是不存在的

端口 port

一种硬件

CPU <–> Port <–> I/O

端口编号就是端口地址，[0000h, 0FFFFh] 65536个端口

绝大部分是空闲的，60h端口与键盘相关，70h、71h与cmos相关

对端口的操作使用指令in和out实现

对端口操作使用指令in与out实现。in从端口中读取信号，out把信号发送给端口
in与out 最底层，不依赖于任何别人写的代码，没有任何调用
汇编语言实现同样的功能，代码有高级、低级之分，最低层次代码威力最大

dos 高级 mov ah, 1 ; int 21h 功能弱，但编程简单
bios 中级 mov ah, 0 ; int 16h
in/out 低级 in al, 60h 功能强，但编程麻烦
