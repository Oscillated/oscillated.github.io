设ss=2000, sp=4000

call 2000 时，cpu会把下条指令的偏移地址即3006压入堆栈；之后cpu再执行jmp 2000 指令。

ret 时，cpu会从堆栈中弹出一个字，并把此字赋值给ip，实现了jmp 3006

ss:3FFE 3006

ss:4000 ??

call相比jmp，跳了以后还能回来，因为跳之前把下一条指令地址压栈保存了，返回时把指令地址赋给ip

32位cpu下call时压入32位地址，ret弹出32位地址

汇编语言中的三种参数传递方式

寄存器传递

变量传递

变量传递的缺点，函数f不能同时被重用(reentrant)，f不能支持多线程

在汇编语言中，用db、dw等关键词定义的变量均为全局变量。在堆栈中定义的变量才是局部变量。

堆栈传递

设ss=1000h，sp=2000h

ss:1FFA old bp <- (3), new bp = sp

ss:1FFC back <- (2) (4)

ss:1FFE 3 <- (1) , [new bp+4] (5)

ss:2000 ?? <- ss:sp (6)

在函数开始时，通过push bp和mov bp, sp可以构造堆栈框架(stack frame)

每个函数都需要一个bp来寻找自己需要的参数，如果g不加bp的保护(构造堆栈框架的两句话)，它返回的f就会混乱

虽然32位cpu支持[esp+常数]的用法，但16位不支持[sp+常数]，只能用[bp+常数]形式

int、iret

int 21h对应的函数首地址（中断向量）保存在0:84h处, 该地址是一个远指针。
0:84 78h
0:85 56h
0:86 34h
0:87 12h
dword ptr 0:[84h]称为int 21h的中断向量(其实是它的函数首地址) 32位
int n对应的中断向量的存储位置一定是0:n*4
n的取值范围是[00, FF], 所以256个中断向量会占用0:0~0:3FF之间共400h个字节的内存，这块区域称为中断向量表。

后128个几乎是空的

int 00h的中断向量保存在dword ptr 0:[0];

int 01h的中断向量保存在dword ptr 0:[4];

int 02h的中断向量保存在dword ptr 0:[8];

int 03h的中断向量保存在dword ptr 0:[0Ch];

int 08h的中断向量保存在dword ptr 0:[20h];
int 0FFh的中断向量保存在dword ptr 0:[3FCh];
BIOS会完成部分中断向量的填写, 如int 10h、int 16h、int 8h, int 9h, int 13h这几个BIOS中断的向量在DOS启动前就已经填好了; DOS启动完成后，会填入int 21h的中断向量。

用户可以修改中断向量，但修改时通常需要保存原来的中断向量以便让代码与旧代码能够链接

例如 int 21的旧中断向量为1234:5678，现在把他保存在old_21h(4字节变量)中，将其改成自己的函数地址如cs:my_int21

设es=0, bx=84h

1000:2000 mov ah, 2
1000:2002 mov dl, ‘A’
1000:2004 int 21h
1000:2006 mov ah,4Ch
上面这条int 21h指令执行时, cpu做了以下4件事:

pushf
push cs即1000h
push 下条指令的偏移地址即2006h
jmp dword ptr 0:[84h]; jmp 1234h:5678h

上面这条jmp会跳转到int 21h的中断服务函数内部:
1234:5678 …
…
1234:56FF iret; 中断返回
iret在执行时, cpu做了以下3件事情:
pop ip即ip=2006h
pop cs即cs=1000h
popf

1000:2000 call 1234:5678; 远调用
;此时会push 1000h 再 push 2005h
;最后jmp 1234:5678
1000:2005 mov ah, 4Ch
1000:2007 int 21h
…
1234:5678 …
retf; 此时执行pop ip再pop cs
