逻辑地址——段地址:偏移地址

  • 物理地址：用一个整数表示的地址称为物理地址

  • 汇编语言的寻址只能使用逻辑地址，不能使用物理地址。

  • 8086CPU中寄存器均为16位，用来表示地址的bx,bp,si,di

  • 为何不让CPU支持直接物理地址的访问？

    • 因为如果使用某个16位寄存器指针bx引用变量值，如果出现较大的地址就无法引用

      • mov bx, 12398h ;错误 超过16位

        1. mov al, [12398h] ; 错误 方括号里只能是16位的值
    1. mov al, [bx] ; 错误

    mov dx, 1000h

    mov ds, dx

    mov al, ds:[2398h] ; 正确

    mov al, 1000h:[2398h] ; 错误，引用某个变量时，段地址不能用常数表示，必须用段寄存器

    mov dx, 1000h

    mov es, dx

    mov bx, 2398h

    mov al, es:[bx] ; 正确 == 1000h:[2398h] == [12398h]

    > 
>
>
    • 解决方法：用两个地址进行拼接，形成20位地址

  • 段起始地址的16进制个位（二进制低4位）必须为0，否则不称为段起始地址

    • 计算段长度

      • 12340h到2233Fh ：2233Fh -12340h + 1 = 22340-12340 = 10000h = 64K=2^6 * 2^10​

        1000h = 4K , 1k = 2^10

        一个段长度不能超过64K 16位

  • 段地址为段首地址去掉末尾0

    • 逻辑地址表达法
    • 一个段segment 每2233F-12340 = FFFF大小的存储空间(64K)
    • 1234:0058 -> 12340h+0058h = 12398h
    • 必须是段寄存器，不能是常数
    • 引用某个变量时，段地址不能用常数表示，必须用段寄存器cs,ds,es,ss之一

  • 偏移地址为0000到FFFF：离开段首的距离

  • 出现重叠现象：一个物理地址可以表示成多个逻辑地址

    • 12398h = 1234:0058 = 1236:0038 = 1230:0098 = …
    • 逻辑地址决定于对于哪一个段首
    • 逻辑地址数量与前面有多少种段首有关

  • 段地址和偏移地址均为16位，段地址:偏移地址可以表示20位的物理地址。8086系统中，地址线一共只有20条，即最多只能寻址1M内存空间，即dos系统只能访问1M内存空间

  • 段地址的1h == 段首地址的10h == 偏移地址的10h

一张128k的图片，需要两个段，需要两个段，填满一个段后，图片只存了一半，因此需要做段切换：设ds = 1234h ,bx = 0, 现要从该地址起填入128k 数据

  1. 把 ds : bx 起共 10000h 全部填满即在1234：0000 ~ 1234：FFFF之间填入64k数据

  2. 接下来修改ds = 2234 : 0 再在2234:0000~2234：FFFF之间填入 64k 数据

    1000:FFFF的下一地址是2000:0000

    编程实现跳到2000:0000再做一次

  • dos中编程，一旦数据超过64k就需要换段，段并非连续的
  • 32位cpu如80386支持32位的偏移地址，可以用byte ptr ds:[ebx]这样的形式来引用4G内存空间中的任何一个字节，ebx [00000000h, FFFFFFFFh]
  • 16位cpu工作在实模式real mode，而32位cpu工作在保护模式protected mode。在实模式下，用户程序具有跟操作系统一样的权限，可以执行任何指令；保护模式下，用户程序的权限低于操作系统，有鞋特权指令无法执行。

  • 除了cs不能通过mov修改，ds、es、ss均可以修改

假定要取出1000:2000指向的字节并保存到AL中，则有两种方法：

  • 直接寻址（用常数来表示变量的偏移地址）

C语言写法：

  • 间接寻址（用寄存器、寄存器+常数来表示变量的偏移地址）
    • [bx] [bx+2] [bx+si] [bx+si+2]
      • 常数可以是负数
      • 但两个寄存器只能加、不能减

    • 确定变量的宽度：当变量与常数在一起做mov, add, sub, or, xor, shr, shl 运算时
      常数没有宽度 既可以当成8位 也可以当场成16位

如何引入数组元素

C写法

C语言与汇编语言在处理short int 元素时不同之处：

  • 汇编中一定要写成xyz[2]，但是C语言中为xyz[1]；汇编中2指的是+2，而不是第二个元素，C语言中1指第二个元素
  • C语言为有类型语言，编译器会自动追踪变量与数据的类型
    • short int xyz[2] = {0x789A,0xBCDE}
    • C语言绝对不会把xyz地址直接加1，而是这个1要乘以一个系数

当C语言编译器遇到xyz[1]时

  1. 首先会获得xyz的类型是short int

  2. 接下来计算 offset xyz + 1*sizeof(short int) C语言中指针加1的实际含义

  3. 获取 word ptr [offset xyz + 1*sizeof(short int)]的值

因为汇编中xyz数组名，2加上数组名才为新的偏移地址 2为相对于首地址元素的增量

汇编中若写为xyz[1]，输出DE78

  • mov ah, [abc]

    • [abc]可以理解成地址abc所指向的对象。
    • 完整形式：mov ah, byte ptr ds:[abc]
      • 其中ds是变量abc的段地址，byte ptr表示地址abc所指的对象是一个字节。
    • 若[]中不包含寄存器bp，则该变量默认的段地址一定是ds，故可以省略ds:。

  • byte ptr类似于C的(char *), 其中ptr是单词pointer的缩写。

    • byte ptr ds:[abc]表示地址ds:abc所指的对象是一个byte
      • 相当于C：*(char *)(ds:abc)

  • 汇编语言的语句中，如果源操作数或目标操作数的其中之一有明确的类型即宽度，则另外一方不需要指定类型。

eg

  • mov ds:[bx], ah ; mov byte ptr ds:[bx],ah ah是8位
  • mov ax, ds:[bx+si] ;mov ax, word ptr ds:[bx+si] ax是16位
  • 当源操作数为常数，目标操作数为变量时，则该变量必须加上类型修饰

eg

  • mov ds:[bx], 1; 语法错误 对 ds:[bx] 必须加以宽度限制
  • mov byte ptr ds:[bx], 1; 正确① 01h
  • mov word ptr ds:[bx], 1; 正确② 0001h 注意小端规则，先填01，后填00
  • mov dword ptr ds:[bx], 1; 正确③ 00000001h
