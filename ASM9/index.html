32位除以16位得16位

dx:ax / 除数 = ax …… dx （冒号表示被除数的左边是高16位在dx，右边是低16位在ax（不是实际程序语句）；除数是操作数寄存器，商在ax，余数在dx）

记忆：小端规则，保存dx和ax时，先存放ax（商），再存放dx（余数）

64位除以32位得32位

edx:eax / 除数 = eax … edx （冒号表示被除数的左边是高32位在edx，右边是低32位在eax（不是实际程序语句）；除数是操作数寄存器，商在eax，余数在edx）

记忆：小端规则，保存edx和eax时，先存放eax（商），再存放edx（余数）

远指针、近指针

远指针 far pointer

16位汇编中，远指针是指16位段地址+16位偏移地址;

32位汇编中，远指针是指16位段地址+32位偏移地址。

48位的远指针在汇编中有一个类型修饰词：fword ptr

近指针 near pointer

16位汇编中，近指针是指16位的偏移地址；

32位汇编中，近指针是指32位的偏移地址；

远指针(far pointer)包括段地址及偏移地址两个部分；

近指针(near pointer)只包括偏移地址，不包含段地址。

32位系统下远指针的段地址为什么没有增加？

在32位系统下，16位的段地址不再通过后面补16进制的0来得到段首地址，而是通过查表得到。

表：全局描述符表Global Descriptor Table，简称gdt表。作为一个数组，其首地址存放在gdtr寄存器内，每个元素宽度均为8字节

t+0h xx,xx,xx,xx,xx,xx,xx,xx ; t[0]

t+8h … ; t[8]

t+10h … ; t[10h]

t+18h xx,xx,40h,30h,20h,xx,xx,10h ; t[18h]

本来ds的取值范围有[0,FFFF]，现在必须是8的倍数（用于查表） 有$2^{16}/2^3=2^{13}$取法

获得18h:12345678h所指的值：

  1. 把段地址18h与t相加，得到gdt表内对应内容（8字节）
  2. 第三个元素的偏移中，第2、3、4、7个字节逆向排列得到段首地址=10203040h
  3. gbt表各元素8字节的其他字节有规定段的长度、段的权限的功能等用途
  4. 段首地址与偏移地址相加
    • 逻辑地址 18h:12345678h
    • 物理地址 10203040h + 12345678h
