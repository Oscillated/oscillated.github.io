Hardware Security Building Blocks

使用内存不安全的高级语言如C和C++开发的系统软件容易受到内存漏洞的Runtime攻击，为了应对这种攻击，防御者想到了跳脱出软件系统的措施——硬件防御。

硬件防御近年来开始流行，向着安全性和成本限制的平衡发展，

硬件保护分为3类：CPU保护、内存保护和I/O保护

CPU Protection

用户无法给CPU一个永久持续的进程，否则其他进程无法执行。为了实现这一功能，系统内部的一个计时器负责给一个进程设置一个时间，时间终止后会使进程强制离开CPU。

Memory Protection

当内存中有两个及以上的进程时，可能会发生一个进程访问其他进程的内存的情况。为了防止这一情况，Bare Register和Limit Register分别用于记录进程的起始地址和其内存大小，当一个进程想要访问内存时，检查它是否能够被允许访问对应的内存空间。

I/O Protection

I/O 保护用于防止内存的以下情况

  • Termination I/O of other process
  • View I/O of other process
  • Giving priority to a particular process I/O

当应用程序进程想要访问任何I/O设备时，都需要通过系统调用，进而由操作系统监视这一调用的安全性。

如C中的write()和read()就是对文件读写的系统调用

  • 在用户模式下 user mode
    • 系统代替应用程序执行这个任务，用户的应用程序无法直接访问硬件及其相关内存
  • 在核模式下 kernel mode
    • 每当需要直接访问硬件时，应用程序就会使用系统调用。
