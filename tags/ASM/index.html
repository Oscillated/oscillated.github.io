    堆栈 ss sp

    ss:stack segment 堆栈段，表示堆栈的段地址

    sp:stack pointer 堆栈指针，表示堆栈顶端的偏移地址（堆栈段长度）

    不能用[sp]或[sp+1][sp+ax]的形式来引用某个变量。

    为了弥补sp不能表示间接地址的缺陷，同时也为了凑齐4个偏移地址寄存器，引入通用寄存器的bx：

    bx, bp, si, di —— 4个可以放在[]中表示变量的偏移地址

    [ax][cx][dx]均是错误的

    寄存器是稀缺资源，因此用堆栈

    push abc[0]的过程

    1. sp = sp - 2 = 200h-2 = 1FEh

    2. 把push后面所跟的值保存到ss:sp当前指向的内存单元中（小端规则）

      • ss:1FE 34h
      • ss:1FF 12h

    pop abc[2]的过程

    1. 把当前ss:sp指向的字取出来，保存到pop后面所跟的变量中
    2. sp = sp+2

    push和pop只能是word或者dword，不允许8位单个字节的数据。

    堆栈主要用于临时保存一些寄存器的值

    任何一个程序均需堆栈，即使没用到push和pop，每隔1/18秒，CPU会产生一个中断int 8，只要int 8产生，就会主动做出push和pop的举动，因为中断的存在，一定需要堆栈。

    如果源程序中未定义堆栈段，dos会自动分配一个堆栈段给exe，ss=首段段址，sp=0

    自动分配的情况下：

    5DC0:0000 psp 长度100h

    5DD0:0000 data 若长度10h

    5DD1:0000 code 若长度20h

    5DD0:0030 stack

    …

    ss=5DD0, sp=0

    push ax; sp=sp-2=FFFE ;5DD0:FFFE

    push bx; sp=sp-2=FFFC ;5DD0:FFFC

    ;push方向向上，堆栈空间5DD0:0030-5DD0:FFFF 一般不会碰到code和上面的段

    当sp=30h时再执行push指令会发生堆栈溢出，只要保证sp>=30h

    psp

    程序刚开始运行时，DOS会把exe文件包含的机器码复制到内存，首段前分配psp。ds=es=psp段址（Program Segment Prefix），再赋值ss:sp=堆栈段址:堆栈长度，再赋值cs:ip=代码段址:main偏移地址。

    psp段址位于程序首个段（代码上体现）的前面，长度100h字节，用途是保存一些exe相关信息。（段地址相差10h，相当于偏移地址相差100h）后面是data code stack（本代码）

    psp:80h 存放程序命令行参数（第0个参数是文件名但80h不存放，首个是参数长度，后面开始第一个参数）

    5DC0:0000 start of psp

    5DC0:00FF end of psp

    5DC0:0100 = 5DD0:0000 start of data segment

    5DD0:0000 34, 12, 78 start of data segment

    5DD0:0003 56 end of data segment

    5DD0:0004 = 5DD04 （不能成为段起始地址，因为最右一位不为0） -> 00, 00, 00 … 填了12个0凑到10h字节

    5DD0:0010 = 5DD10 start of code segment

    …

    （补零）

    5DD0:0030 = 5DD3:0000 start of stack segment

    32位的堆栈

    push/pop 后面可以跟16位的寄存器或变量（8086），后面也可以跟32位的寄存器、变量、常数（80386）

    设ss=1000h, sp=2000h

    mov eax, 12345678h

    push eax ; sp = sp - 4, ss:[sp] = 12345678h

    1000:1FFC 78h

    1000:1FFD 56h

    1000:1FFE 34h

    1000:1FFF 12h

    pop ebx ; ebx = ss:[sp]=12345678h, sp = sp + 4

    附加段 es

    es：extra segment 附加段，类似ds，可以用来表示一个数据段的段址

    寄存器小结

    ax bx cx dx 计算，bx还可以用来表示偏移地址

    sp bp si di 偏移地址，除了sp另外3个也可参与计算

    cs ds es ss 段地址

    ip fl

    cs:ip指向当前将要执行的指令

    ss:sp指向堆栈顶端

    标志寄存器 FL

    16位，只用其中9位 6个状态标志+3个控制标志，不能用于加减乘除运算

    最右端第0位CF:carry flag 进位标志

    CF=1表示出现了进位或借位

    左移及右移出去的1位会填到CF中

    获得CF当前状态的方法

    cmp cf, 1; 语法错误

    专门指令：

    clc:使CF=0 clear carry flag

    stc:使CF=1 set carry flag

    cmc:使CF反转 complement carry flag

    jb: jump if below

    jc: jump if carry flag

    jb、jc两条指令是等价的

    jc jnc je jz jne jnz jb ja 都是根据当前fl寄存器中标志位的状态做出判断和跳转的。

    会影响标志位的指令(add sub cmp等)后面都可以接以上指令

    ptr

    部分见4.

    另外还有两种ptr用来修饰标号

    near ptr 近，当前指令与目标地址在同一个段内

    far ptr 远，当前指令与目标地址不在同一个段内

    near ptr 和 far ptr的前面通常是jmp或call

    • jmp near ptr next
    • jmp far ptr away

    标号 —— main: next: code段内

    变量名 —— 无冒号 data段内

    当目标操作数与源操作数的其中之一有明确的宽度时，另外一方不需要加类型修饰

    当源操作数为常数，目标操作数为变量时，则该变量必须加上类型修饰

    mov ds:[bx], 1; 错误
    mov byte ptr ds:[bx], 1; 正确
    mov word ptr ds:[bx], 1; 正确
    mov dword ptr ds:[bx], 1; 正确

    图形模式

    通过写入B800段内的显卡内存来控制屏幕字符输出是文本模式编程

    映射机制mapping：把显卡内存和RAM中的B800段建立对应关系。

    A000段开始到F000段均不是用户的内存空间，部分内存映射到显卡内存中，部分内存映射到ROM中。

    POST(Power on self test)位于ROM中，映射到F000段中，只读、不能更改

    DOS中留给用户的内存空间只有0000:0000-9000:FFFF 640k内存

    • 显卡有两种模式 图像模式G（13h） 文本模式T(14h)

    逻辑地址——段地址:偏移地址

    • 物理地址：用一个整数表示的地址称为物理地址

    • 汇编语言的寻址只能使用逻辑地址，不能使用物理地址。

    • 8086CPU中寄存器均为16位，用来表示地址的bx,bp,si,di

    • 为何不让CPU支持直接物理地址的访问？

      • 因为如果使用某个16位寄存器指针bx引用变量值，如果出现较大的地址就无法引用

        • mov bx, 12398h ;错误 超过16位

          1. mov al, [12398h] ; 错误 方括号里只能是16位的值
      1. mov al, [bx] ; 错误

      mov dx, 1000h

      mov ds, dx

      mov al, ds:[2398h] ; 正确

      mov al, 1000h:[2398h] ; 错误，引用某个变量时，段地址不能用常数表示，必须用段寄存器

      mov dx, 1000h

      mov es, dx

      mov bx, 2398h

      mov al, es:[bx] ; 正确 == 1000h:[2398h] == [12398h]

      > 
>
>
      • 解决方法：用两个地址进行拼接，形成20位地址

    • 段起始地址的16进制个位（二进制低4位）必须为0，否则不称为段起始地址

      • 计算段长度

        • 12340h到2233Fh ：2233Fh -12340h + 1 = 22340-12340 = 10000h = 64K=2^6 * 2^10​

          1000h = 4K , 1k = 2^10

          一个段长度不能超过64K 16位

    • 段地址为段首地址去掉末尾0
      • 逻辑地址表达法
      • 一个段segment 每2233F-12340 = FFFF大小的存储空间(64K)
      • 1234:0058 -> 12340h+0058h = 12398h
      • 必须是段寄存器，不能是常数
      • 引用某个变量时，段地址不能用常数表示，必须用段寄存器cs,ds,es,ss之一
    • 偏移地址为0000到FFFF：离开段首的距离
    • 出现重叠现象：一个物理地址可以表示成多个逻辑地址
      • 12398h = 1234:0058 = 1236:0038 = 1230:0098 = …
      • 逻辑地址决定于对于哪一个段首
      • 逻辑地址数量与前面有多少种段首有关
    • 段地址和偏移地址均为16位，段地址:偏移地址可以表示20位的物理地址。8086系统中，地址线一共只有20条，即最多只能寻址1M内存空间，即dos系统只能访问1M内存空间
    • 段地址的1h == 段首地址的10h == 偏移地址的10h

    一张128k的图片，需要两个段，需要两个段，填满一个段后，图片只存了一半，因此需要做段切换：设ds = 1234h ,bx = 0, 现要从该地址起填入128k 数据

    1. 把 ds : bx 起共 10000h 全部填满即在1234：0000 ~ 1234：FFFF之间填入64k数据

    2. 接下来修改ds = 2234 : 0 再在2234:0000~2234：FFFF之间填入 64k 数据

      1000:FFFF的下一地址是2000:0000

      编程实现跳到2000:0000再做一次

    • dos中编程，一旦数据超过64k就需要换段，段并非连续的
    • 32位cpu如80386支持32位的偏移地址，可以用byte ptr ds:[ebx]这样的形式来引用4G内存空间中的任何一个字节，ebx [00000000h, FFFFFFFFh]
    • 16位cpu工作在实模式real mode，而32位cpu工作在保护模式protected mode。在实模式下，用户程序具有跟操作系统一样的权限，可以执行任何指令；保护模式下，用户程序的权限低于操作系统，有鞋特权指令无法执行。

    如何引入数组元素

    C写法

    C语言与汇编语言在处理short int 元素时不同之处：

    • 汇编中一定要写成xyz[2]，但是C语言中为xyz[1]；汇编中2指的是+2，而不是第二个元素，C语言中1指第二个元素
    • C语言为有类型语言，编译器会自动追踪变量与数据的类型
      • short int xyz[2] = {0x789A,0xBCDE}
      • C语言绝对不会把xyz地址直接加1，而是这个1要乘以一个系数

    当C语言编译器遇到xyz[1]时

    1. 首先会获得xyz的类型是short int

    2. 接下来计算 offset xyz + 1*sizeof(short int) C语言中指针加1的实际含义

    3. 获取 word ptr [offset xyz + 1*sizeof(short int)]的值

    因为汇编中xyz数组名，2加上数组名才为新的偏移地址 2为相对于首地址元素的增量

    汇编中若写为xyz[1]，输出DE78

    • mov ah, [abc]

      • [abc]可以理解成地址abc所指向的对象。
      • 完整形式：mov ah, byte ptr ds:[abc]
        • 其中ds是变量abc的段地址，byte ptr表示地址abc所指的对象是一个字节。
      • 若[]中不包含寄存器bp，则该变量默认的段地址一定是ds，故可以省略ds:。

    • byte ptr类似于C的(char *), 其中ptr是单词pointer的缩写。

      • byte ptr ds:[abc]表示地址ds:abc所指的对象是一个byte
        • 相当于C：*(char *)(ds:abc)

    • 汇编语言的语句中，如果源操作数或目标操作数的其中之一有明确的类型即宽度，则另外一方不需要指定类型。

    eg

    • mov ds:[bx], ah ; mov byte ptr ds:[bx],ah ah是8位
    • mov ax, ds:[bx+si] ;mov ax, word ptr ds:[bx+si] ax是16位
    • 当源操作数为常数，目标操作数为变量时，则该变量必须加上类型修饰

    eg

    • mov ds:[bx], 1; 语法错误 对 ds:[bx] 必须加以宽度限制
    • mov byte ptr ds:[bx], 1; 正确① 01h
    • mov word ptr ds:[bx], 1; 正确② 0001h 注意小端规则，先填01，后填00
    • mov dword ptr ds:[bx], 1; 正确③ 00000001h

    屏幕输出字符”$”汇编语言代码分析

    • 首先将ds值赋为data段地址，因为只要程序中引用了data段内的变量，则必须在程序一开始就把ds赋值为data地址未对ds进行赋值，ds内的值一定不等于data，不等于数据的段地址
      • move ax, data
      • move ds, ax
      • 保证ds = data
      • 可用seg s替换data，更符合数据“所在的段”的理解
      • 可以用es（附加段寄存器）代替ds（数据段寄存器），使用ds时ds:可以被隐藏，但是被替代后必须显式存在
    • (ds:)s[bx]
      • ds - 段地址
      • s - 数组首地址/数组名
        • C中其实也可实现数组名放在[]内的写法: 0[a+i] = *(0+a+i)
      • bx - offset address
      • 编译后会将数组名s转换成该数组的偏移地址
      • s[bx] -> 3[bx] -> [bx+3]
        • 依次是操作码opcode 目标操作数object operand 源操作数source operand
      • 编译后：mov dl, [bx+3]
      • 操作数是指令的参数
      • 汇编中出现一对方括号可认为里面是变量
      • C写法
        • p = ds:(bx+3)
        • dl = *p
      • 查看ds:0是先前的字符串ABC
    • 调用int21h的9号功能输出字符串时，字符串中间出现$号，之后内容就会截断（也不输出$）
      • 要想输出$，使用int21h的2号功能
      • 通过循环调用
      • 每调用一次输出一个字符，将字符串理解成字符
      • 使用的寄存器是dl,ah
      • s[bx]写法类似C的数组下标元素使用，但括号内必须是一个寄存器名字，不能是变量名
      • 使用bx前需要初始化 mov bx, 0
    • 退出函数
      • mov ah, 4Ch
      • (mov al, 0)
      • int 21h
      • 相当于exit(al)
      • 括号行可省略，因为操作系统会将随机al值丢弃
    • 如何知道数组s的偏移地址offset（s离开该段段首的偏移距离）
      • 即s之前字符串的长度和
    • assume的理解
      • 只是假设，并不会对后面的寄存器赋值，需要手工赋值。
      • 编译器也不会校验es是否真的等于data。

    小端规则、大端规则

    宽度超过8位就会被切

    Little-Endian 先存放低8位 后存放高8位

    小端适用于Intel的CPU型号 大端少见

    address value(small) value(big)
    1000 0x78 0x12
    1001 0x56 0x34
    1002 0x34 0x56
    1003 0x12 0x78

    C的指针与小端规则

    设&a = 1000，于是p = 1000。p是char类型指针，只能指向1000所在的0x34。&a是short int*类型，值为1000，但指向的是1001 0x12和1000 0x34

    设置成小端规则是为了处理数据截断时与实际情况的一致

    8位 16位 32位整数的取值范围

    补码中表示N位最大值，最高位为0，后面为连续N-1个1；最小值，最高位1，后面为连续N-1个0。

    16 unsigned

    [0000h, 0FFFFh] [0, 65535]

    32 unsigned

    [0, 2^32-1]

    8 signed

    16 signed

    [8000h, 7FFFh] [-32768, 32767]

    32 signed

    • 正数左端补0

    汇编语言中寄存器名字的含义

    4个16位段寄存器

    给出相应逻辑段的首地址，称段基址。段基址与偏移地址组合形成20位物理地址，偏移地址可以存放在寄存器中，也可以存放在存储器中。

    cs和ss是专用的，无需做任何赋值，操作系统自动为cs和ds赋值。

    ds和es需要手工赋值。

    CS

    代码段寄存器，代码段段地址

    DS

    数据段寄存器

    SS

    堆栈段寄存器，等于堆栈段址

    ES

    附加段寄存器，对数据段的补充

    AX

    累加器，一些指令约定以AX AL AH为源或目的，输入输出指令必须通过AX/AL实现。

    传递参数/返回值，固定存储某些值

    BX

    基址寄存器，可用作间接寻址的地址寄存器和基地址寄存器，BH,BL可用作8位通用数据寄存器。

    寻址

    CX

    计数寄存器，在循环和串操作中充当计数器，指令执行后CX自动修改。

    控制循环次数

    DX

    数据寄存器，用作通用寄存器，在I/O指令中可用作端口地址寄存器，惩处指令中用作辅助累加器。

    传递参数，固定存储某些值

    BP

    base pointer register，基址指针寄存器

    SP

    stack pointer register，堆栈指针寄存器

    SI

    source index register，源变址寄存器

    寻址

    DI

    destination index register，目的变址寄存器

    寻址

    IP

    指令指针寄存器
